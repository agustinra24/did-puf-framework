ESP32 SRAM-PUF Provisioning & Firmware Configuration

ESP32-WROOM-32D · ESPPufLib · Web Serial API · v2.0
Prerequisitos: Usa Google Chrome o Microsoft Edge en escritorio. Conecta el ESP32 via USB. Para el diagnostico, presiona BOOT al conectar si el firmware esta corriendo. El flasheo con erase-first se conecta automaticamente.
1

Diagnostico del Chip

Conecta al ESP32 para leer su informacion de hardware: modelo, revision, tamano de flash, MAC address y crystal. Esto verifica que el chip es compatible con el enrollment de SRAM-PUF.

2

Instalar Firmware de Enrollment

Flashea el firmware de SRAM-PUF enrollment al ESP32. El firmware realizara multiples ciclos de deep-sleep para capturar el estado de la SRAM, extraer bits estables y generar el PUF Response unico del chip. Este proceso puede tomar entre 1 y 6 minutos segun la cantidad de mediciones configuradas.

Incluye erase-flash automatico
3

Monitor Serial — Enrollment

Observa en tiempo real los ciclos de deep-sleep y la extraccion del PUF Response. Busca el mensaje PROVISIONING DONE seguido del volcado hexadecimal del PUF Response.

Pega el PUF Response (hex) de dos chips diferentes para calcular la Hamming Distance. Se espera ~50% entre chips distintos (independencia estadistica) y ~0% para lecturas repetidas del mismo chip sin re-enrollment (reproducibilidad). Nota: re-enrollments del mismo chip producen respuestas diferentes porque el erase-flash destruye el helper data y se seleccionan nuevos bits estables.

1

Configuracion del Dispositivo

Define los parametros de conexion del dispositivo. Estos se enviaran al ESP32 via serial despues del flasheo. El firmware los almacena en NVS (Non-Volatile Storage) y los usa para conectarse al servidor IoT.

Servidor IoT
WiFi
2

Instalar Firmware Funcional

Flashea el firmware operativo que conecta al servidor IoT. Este firmware incluye el cliente HTTP, la lectura de NVS para parametros, y el reporte periodico. No borra la flash para evitar destruir datos NVS existentes. El PUF Response se transfiere via serial en el Paso 3.

Sin erase (preserva NVS existente)
3

Enviar Configuracion al Dispositivo

Envia los parametros de conexion y el PUF Response capturado en la Pestaña 1 al ESP32 via serial. El firmware entra en modo configuracion automaticamente si detecta que no tiene parametros almacenados. Protocolo: CFG_START → pares CLAVE=valor + PUF_RESPONSE → CFG_END.

PUF: —
4

Verificar Conexion

Monitorea la salida serial del ESP32 para confirmar que se conecta al WiFi y reporta exitosamente al servidor IoT. Busca mensajes como WiFi connected y HTTP 200.
